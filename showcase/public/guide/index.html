The Guide

How nine worlds were drawn in code.

This page is the honest making-of. Everything in the gallery — the shaders, the flocks, the rain, the botany, the charts — was designed and built by Claude, working autonomously from a single commission, under one hard rule and one uncompromising process. Here is exactly how, so you can do it too.

The one rule: zero assets

If it appears on screen, code drew it. No photographs, no stock imagery, no AI-image or AI-video pipelines, no downloaded textures, no icon packs, no CDNs, and no runtime network requests of any kind. Each room is plain HTML, CSS, and JavaScript served statically. The permitted binaries: self-hosted font files, and the gallery thumbnails — which are screenshots of the rooms themselves.

The rule is not an aesthetic handicap; it is the demonstration. A smoke shader you wrote behaves; a stock video of smoke merely plays. Every room's signature visual is therefore also its signature interaction — the smoke stirs around your pointer, the flock flees your cursor, the fog wipes and re-condenses.

The stack, deliberately boring

Static HTML + CSS + vanilla JavaScript per room; no frameworks and no build step. One room (LYRA) imports three.js r160 as an ES module, vendored locally. Fonts are Google Fonts fetched once at build time by a small script and self-hosted as woff2 under /fonts/ — 38 families, zero runtime requests. Serving is any static host; there is a netlify.toml with cache headers in the repository.

The interesting engineering lives inside the pages: raw-WebGL fragment shaders (domain-warped fBm for smoke, ink, and light fields), canvas 2D simulations (boids with a spatial hash, rain droplets with lens refraction, zoned ocean life), generative SVG (L-system-style botany from seeded grammars, hand-built charts, stroke-drawn line art), and CSS used as an optical instrument (the arcade's CRT is layered scanlines, chromatic text-shadows, and curved-glass gradients).

The process

  1. A binding charter. One document set the bar for every room: code-drawn visuals only; real, original copy for a fictional brand; a "wow" inside three seconds; one discoverable signature interaction; custom easing; layered atmosphere; a favicon drawn in SVG; and an engineering floor — capped device-pixel-ratio, animation loops that pause offscreen, prefers-reduced-motion honored as a first-class rendering, semantic markup, mobile at 390px with no horizontal scroll.
  2. An art-direction brief per room. Before any code: the fictional brand, the palette in hex, the exact type pairing, the section rhythm, the signature technique, and the interaction. Nine briefs, no two sharing a palette, typeface set, or core technique.
  3. Parallel builders, one art director. Several rooms were built by parallel instances of Claude, each handed the charter and one brief; the flagship rooms were built directly. Every room, whoever built it, was screenshot-reviewed and gated centrally before it counted.
  4. Three iteration passes, with eyes. After a room's first complete draft: screenshot every section at 1440×900 and 390×844 with headless Chromium, actually look at the images, write down the faults, fix them, re-shoot. Pass one hunts composition and hierarchy; pass two deepens color, motion, and richness; pass three is polish — mobile, reduced motion, alignment nits. The bugs this caught ranged from "the nebulae are too faint" to "the droplet trails read as worms" to a crashed constructor that blanked half a page.
  5. A hard QA gate. Zero console errors; zero non-local requests (asserted with a network listener); a complete, beautiful page under reduced motion; no horizontal scroll at 390px; all links resolving. No room shipped without passing.

The rooms, technically

02 ÉCLATDomain-warped fBm smoke in monochrome; a pointer-following field adds a thin-film interference hue term, so color exists only where the visitor moves. The flacon is hand-plotted SVG that draws itself.
03 SOUNDINGScroll maps to depth (0 → 10,935 m) through a piecewise color ramp; sea life is zoned and procedural — boid fish, bezier jellyfish, marine snow — with click-sonar that reveals hidden creatures.

05 FOLIUMA generative botany engine grows seven vector species from seeded grammars — tapering stem chains, unfurling organs — plus a propagation plate; the same seed regrows the same plant, leaf for leaf.
08 MIDNIGHT COINA CRT built from CSS optics (scanlines, RGB fringe, curved-glass sheen, rare tracking glitches), six generative attract-mode marquees on tiny canvases, and a playable coin-drop game with 2D physics.

Do it yourself

  1. Write the charter first. Constraints are the style. Decide what is banned (assets, lorem ipsum, dead buttons) and what is mandatory (reduced motion, a signature interaction, a favicon) before designing anything.
  2. Give every piece its own world. One palette, one type pairing, one technique per site — written down before the first line of code. If two pieces could swap stylesheets, they are the same piece.
  3. Iterate with your eyes, not your hopes. Screenshot the real page, look hard, write the faults down, fix, re-shoot. Three passes minimum. The screenshot always disagrees with your imagination somewhere, and it is always right.
  4. Treat reduced motion as a rendering, not a removal. A still frame of the shader, the letter already typed, the plant already grown — complete and composed, never blank.
  5. Gate with a script. Console clean, network silent, 390px unbroken, links resolving. Taste decides what ships; the gate decides what may ask.

An honest colophon

This collection was commissioned as twenty-five rooms; the commissioner narrowed it to the finished nine mid-flight, and three additional rooms remain half-built in the repository, preserved but unlisted. The build was interrupted twice by usage limits and resumed from disk both times. Nothing here is a template, a theme, or a component library — every room began as an empty file.
